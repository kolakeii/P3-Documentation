Matching Game

I was inspired by memory matching games, where the player would have to pick out of a set of identical tiles to reveal the item. The goal was

to match all with their identical pair hidden somewhere else on the grid, and memorization of previously discovered items was crucial to succeding.

This seemed perfect for an interactive experience based on showing and hiding interaction functions in js and html

Stage 1 - UI Setup

This project began with the 2x3 grid the user would be playing on, in order to simplify the game to only work with thre color pairs. Each cell would

have an "item" class which defined its size and color, as well as a unique id tag so each could later be selected. From here onward, the css code does not change.

Stage 2 - Assigning Colors

Next each of the cells would have to be randomly assigned one of three colors: red blue and green. However, it could only assign each color

exactly twice, and without overwriting an already assigned cell.


This was done by creating a for loop within the "initGame" function which, upon game initialization, would run through each cell via their

unique id number, and randomly assign them one of the three colors. A array variable called "assigned" was created to keep track of how many

were assigned with each color, and if already two have bee assigned to a color, it would be rerolled until a free color was assigned

Stage 3 - Toggeling

Now, each cell needed to have a toggle function, where upon clicking the cell, it would toggle itself between shown and hidden. This was relegated

to two functions, which were called in the "toggleSquare" function and could be called in future functions. In order to prevent the user from selecting

more than two, the toggle function kept track of how many open squares there were, and prevented itself from toggeling more cells.

Stage 4 - Resolving, Matching and Timing

Next, Logic needed to be introduced which could detect whether two cells which were revealed had identical colors, and if they matched, the pair

would be marked "resolved". If they did not match, the two squares would stay for a period of around 1 second, where no other cell could be

selected, before hiding both simultaneaously. Resolved pairs could also not be toggled off again and when all were resolved, the game would restart.


First, a function was created called "checkForMatchesAndMarkResolved". When the user selects two cells, their color value is saved in the 'openones' array.

Later, if the two values are equal, both are marked in the 'resolved' array, and can no longer be selected in the "toggleSquare" function. Otherwise, the

'hideTwoSquares' function is enabled after around one second, and using the IDs, both images are hidden. Finally the 'sum' counter keeps track of how many

cells are resolved. If all are resolved, the game is re-initialized after about 1 second, beginning the game over again.
